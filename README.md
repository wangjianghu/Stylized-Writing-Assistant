# VS Code 风格化写作助手

一款深度集成于 VS Code 的智能写作辅助插件，帮助用户自动学习写作风格，生成风格一致的文章。

## 核心功能

### 1. 风格学习（14维度分析）
- 自动分析历史文章的写作风格
- 采用14维度专业分析框架：
  - 核心维度：词汇、句式、语气、结构、修辞
  - 深度维度：时间、空间、逻辑、密度、情感
  - 语境维度：受众、文化、叙事、节奏
- 支持工作区多风格档案（可新建、切换、重命名、删除）

### 2. 风格化文章生成（核心功能）
- 输入指定主题，自动生成风格一致的文章
- 支持选择“完整文章”或“写作提纲”（提纲默认输出 2 套以上不同思路）
- 生成结果落地到可配置存储目录下的「文集」Markdown 文件
- 默认使用统一 API 生成；可切换为 IDE Chat 模式并自动填充提示词
- 集成 Humanizer-zh 思路的去 AI 味后处理（弱化机械模板拼接）

### 3. 素材智能管理
- 自动提取精彩句子、优秀段落、引用名言等
- 智能分类和标签管理
- 多维度检索：关键词 + 标签 + 类型过滤
- 支持当前风格素材一键导出 Markdown
- 素材按当前风格隔离存储与检索
- 风格/素材/文集索引持久化到可配置存储目录（默认 `~/Downloads/writing-angent`）

### 4. IDE 深度融合
- Activity Bar 主入口：写作助手容器（素材库、风格配置）
- Explorer 兼容兜底入口：素材库、风格配置
- 快捷入口视图：导入样本数据、根据主题写作、搜索素材、打开文集、补全触发字数
- 若 AI 结果仅出现在聊天窗口，可用“从剪贴板写入当前文稿”一键落盘
- 欢迎入口 + 标题栏按钮（导入、搜索、分析、刷新、AI 续写）
- 命令面板、快捷键、侧边栏入口保持一致

## 快速开始

### 安装依赖

```bash
npm install
```

### 编译

```bash
npm run compile
```

### 调试

1. 在 VS Code 中打开项目
2. 按 F5 启动调试
3. 在新窗口中测试插件功能（默认会打开当前工程工作区，而不是空白欢迎页）

## 使用方法

### 1. 导入样本数据学习风格

```
1. 按 Cmd+Shift+I (Mac) 或 Ctrl+Shift+I (Windows/Linux)
2. 选择历史文章文件（支持 .txt/.md/.doc/.docx，多选）
3. 插件自动分析风格特征并提取素材
4. 查看导入报告
```

### 2. 生成风格化文章

```
1. 按 Cmd+Shift+G (Mac) 或 Ctrl+Shift+G (Windows/Linux)
2. 输入写作主题
3. 选择输出类型（完整文章 / 写作提纲）
4. 等待生成完成，自动在文集创建文稿并打开编辑器
5. 若使用 IDE Chat 模式，提示词会自动尝试填充到聊天输入框
6. 若 AI 内容只在聊天框，复制后执行“写作助手: 从剪贴板写入当前文稿”
```

### 3. 分析当前文档风格

```
1. 打开要分析的文档
2. 按 Cmd+Shift+A (Mac) 或 Ctrl+Shift+A (Windows/Linux)
3. 查看风格分析报告
```

### 4. 搜索素材（独立搜索页）

```
1. 先确认已激活目标风格（可在“风格配置”里切换）
2. 按 Cmd+Shift+S (Mac) 或 Ctrl+Shift+S (Windows/Linux)
3. 输入关键词，按类型勾选过滤条件
4. 在结果列表中查看详情
5. 可直接复制素材或插入到当前编辑器
```

### 5. 侧边栏入口

```
1. 优先在 Activity Bar 点击“写作助手”容器图标
2. 若宿主兼容层不显示容器，按 Cmd/Ctrl + Shift + P 执行“写作助手: 显示写作助手”
3. 在“快捷入口”直接使用“导入样本数据 / 根据主题写作 / 搜索素材 / 打开文集”
4. 在“文集”查看并打开已生成文稿（可直接编辑）
5. 若需要把聊天输出落盘，执行“从剪贴板写入当前文稿”
6. 在“素材库”执行刷新时，会先扫描文集并补齐缺失索引，再沉淀素材
7. 在“风格配置”查看当前风格摘要和风格列表
8. 使用“补全触发字数”快速设置自动补全最小触发字符数（2~20）
9. 使用视图标题栏按钮快速新建/切换/重命名/删除风格
```

### 6. 多风格管理（命令面板）

```
1. 按 Cmd/Ctrl + Shift + P 打开命令面板
2. 输入“写作助手”或“写作”后使用：
   - 写作助手: 显示写作助手
   - 写作助手: 新建风格
   - 写作助手: 切换风格
   - 写作助手: 重命名风格
   - 写作助手: 删除风格
   - 写作助手: 导出素材
   - 写作助手: 从剪贴板写入当前文稿
   - 写作助手: 设置自动补全最小触发字符数
3. 生成文章默认使用“当前风格”
```

| `Cmd/Ctrl + Shift + G` | 生成风格化文章 |

| `Cmd/Ctrl + Shift + S` | 搜索素材 |

## 配置选项

在 VS Code 设置中搜索 "写作助手" 进行配置：

- `writingAgent.autoExtract`: 自动提取素材（默认：true）
- `writingAgent.suggestionDelay`: 建议延迟时间（默认：500ms）
- `writingAgent.suggestionMinChars`: 自动补全最小触发字符数（默认：6，范围：2~20）
- `writingAgent.storage.path`: 插件内容存储根目录（留空时默认 `~/Downloads/writing-angent`）
- `writingAgent.ai.channel`: AI 通道（`api` 默认 / `ide-chat`）
- 也可通过命令“写作助手: 设置自动补全最小触发字符数”快速修改 `writingAgent.suggestionMinChars`

## 技术架构

- **开发语言**: TypeScript
- **框架**: VS Code Extension API
- **存储**: 外部目录 JSON + 文集 Markdown（可配置路径）
- **NLP / 文档解析**: 内置轻量分词 + 可选 nodejieba 降级，mammoth（DOCX 解析）
- **UI**: VS Code TreeView + Webview API

## 手工验收矩阵（Cursor + CodeArts）

### 验收 1：导入命令无 `command not found`

1. 在 Cursor 和 CodeArts 分别执行：`Developer: Reload Window`
2. 在各自 IDE 按 `F5` 启动扩展宿主
3. 执行“写作助手: 导入样本数据”
4. 期望：不出现 `command not found`

### 验收 2：搜索链路可用

1. 在扩展宿主中按 `Cmd/Ctrl + Shift + S`
2. 输入关键词并勾选类型过滤
3. 在结果中分别测试“复制/插入”
4. 期望：筛选结果正确，复制与插入成功

### 验收 3：素材按风格隔离

1. 创建并切换到 `风格A`，导入样本数据后记录一条关键词（如 `A关键词`）
2. 切换到 `风格B`，导入不同文章，记录另一关键词（如 `B关键词`）
3. 在 `风格A` 下搜索 `B关键词`，再在 `风格B` 下搜索 `A关键词`
4. 期望：不同风格之间素材互不可见
